## ADDED Requirements

### 需求:正式纹理交付物必须为 PNG
Sekiro 正式纹理导出必须输出 PNG，并以统一命名和目录规则交付。正式链路禁止把 DDS 作为 PNG 失败时的替代成功结果。

#### 场景:纹理导出成功
- **当** 导出器正式导出角色纹理
- **那么** 输出目录中每个成功纹理都必须是 PNG 文件

### 需求:源纹理解码必须由导出器承担
正式纹理链路必须在导出阶段完成从游戏源纹理到正式 PNG 的解码与像素归一化。下游导入器不得被要求理解 DDS 变体、BC 压缩差异或源容器细节。

#### 场景:导出压缩纹理
- **当** 角色纹理来源于 BC 压缩 DDS 数据
- **那么** 导出器必须生成与正式像素语义一致的 PNG，或者显式报告该格式当前不受支持

### 需求:不支持的源格式必须显式失败
对尚未纳入正式支持矩阵的源纹理格式，导出器必须显式失败并报告源格式、纹理名和角色名，禁止静默跳过或回退转存其他格式。

#### 场景:遇到未支持的源纹理格式
- **当** 导出器遇到当前版本不支持的源纹理编码
- **那么** 当前纹理必须被记录为失败，且该角色不得被记为纹理导出成功
