- **SekiroExporter 可执行文件**: `E:\Sekiro\DSAnimStudio\SekiroExporter\bin\Release\net6.0-windows\SekiroExporter.exe`
- **导出输出目录**: `E:\Sekiro\Export`

## 新增需求

### 需求:CLI批量导出工具
系统必须提供一个独立的命令行工具（SekiroExporter），能从 Sekiro 游戏目录批量导出全部角色资产。

#### 场景:正式链路禁止临时实现
- **当** CLI 执行正式导出链路
- **那么** 角色装配、部件解析、动画交付、材质清单和参数引用都必须来自游戏文件、参数表、规范配置或明确格式契约；不得通过猜测、硬编码角色特例、临时补丁、伪造载体或导出后修补把结果伪装成正式成功

#### 场景:全量导出单个角色
- **当** 用户执行 `SekiroExporter export-all --game-dir "<Sekiro路径>" --chr c0000 --output ./Export/`
- **那么** 系统必须自动从游戏目录定位并加载该角色的 `.chrbnd.dcx`（模型）、`.anibnd.dcx`（动画+TAE）、相关 `.tpf.dcx`（纹理），并导出完整的正式模型资产、全部正式动画资产、全部纹理（PNG/DDS）、技能配置（JSON）和材质清单（JSON）到指定输出目录；只有可被 UE 正式消费的资产才计入完成结果

#### 场景:玩家角色部件必须按正式数据源解析
- **当** CLI 导出 `c0000` 等需要多部件组装的角色
- **那么** 系统必须根据规范配置和正式参数表解析 Head/Face/Body/Arms/Legs/Weapon/义手等部件的真实 `partsbnd` 文件名；对 `c0000` 这类没有单一 `.chrbnd` 成品外观的角色，必须使用仓库内受规范约束的 canonical export profile，禁止使用猜测出来的默认部件列表或仅凭文件存在性拼凑角色

#### 场景:批量导出全部角色
- **当** 用户执行 `SekiroExporter export-all --game-dir "<Sekiro路径>" --output ./Export/` （不指定 --chr）
- **那么** 系统必须扫描游戏目录 `chr/` 下的全部 `.chrbnd.dcx` 文件，对每个角色执行完整导出。独立角色的导出必须使用并行处理以提高效率

#### 场景:选择性导出
- **当** 用户使用子命令指定导出类型（`export-model`、`export-anims`、`export-textures`、`export-skills`）
- **那么** 系统必须仅执行指定类型的导出操作，跳过其他类型

### 需求:导出目录结构
CLI 工具必须按照规范的目录结构组织导出文件。

#### 场景:按角色组织输出
- **当** 导出完成
- **那么** 输出目录必须按照以下结构组织：`<output>/<chrID>/Model/`、`<output>/<chrID>/Animations/`、`<output>/<chrID>/Textures/`、`<output>/<chrID>/Skills/`；其中 Model 和 Animations 目录默认只保留正式交付格式，`material_manifest.json` 和 `skill_config.json` 必须与正式导出资产一一对应

#### 场景:中间文件默认不进入最终导出
- **当** 用户未显式指定调试参数（如 `--keep-intermediates`）
- **那么** CLI 不得默认保留 DAE、重复的 glTF/GLB、中间合并产物或其他调试文件；这些文件不得计入最终导出体积和文件数

#### 场景:导出清单反映完整性
- **当** 一个角色导出结束
- **那么** CLI 必须输出或写入角色级导出清单，明确记录模型、动画、纹理、技能配置、参数表和材质清单是否全部成功，以及最终采用的正式交付格式

### 需求:错误处理和日志
CLI 工具必须提供完整的错误处理和进度报告。

#### 场景:处理损坏的游戏文件
- **当** 某个角色的游戏文件无法正确解析
- **那么** 系统必须记录错误信息（包含角色 ID 和文件路径），跳过该角色并继续处理剩余角色

#### 场景:存在不完整导出时的退出状态
- **当** 某个角色缺少模型、动画、纹理、技能配置或参数表中的任意正式交付物
- **那么** CLI 必须在控制台和最终摘要中明确标记该角色失败，批量导出的最终状态不得报告“全部成功”

#### 场景:进度报告
- **当** 批量导出过程中
- **那么** CLI 必须在控制台输出当前处理的角色 ID、已完成数量/总数量、以及每个导出步骤（模型/动画/纹理/技能）的完成状态

### 需求:项目集成
SekiroExporter 必须作为独立的控制台项目集成到现有解决方案中。

#### 场景:解决方案集成
- **当** 构建 DSAnimStudioNETCore.sln
- **那么** SekiroExporter 项目必须包含在解决方案中，目标框架为 net6.0-windows，引用 SoulsFormats 和 SoulsAssetPipeline 项目，以及 AssimpNet、Pfim、Newtonsoft.Json NuGet 包

### 需求:角色级正式复刻资产包摘要
SekiroExporter 必须为每个角色输出一份可供 UE 复刻链路消费的正式资产包摘要，明确记录资产完整性和关联关系。

#### 场景:导出角色正式资产包摘要
- **当** 用户执行单角色或批量角色导出
- **那么** 系统必须为每个角色输出正式摘要，至少记录模型交付物、动画交付物、PNG 纹理、材质清单、技能配置、参数关联摘要、root-motion 完整性以及这些文件的稳定相对路径和完成状态

### 需求:复刻输入缺失时角色导出失败
SekiroExporter 必须把 UE 复刻所需的关键输入视为正式交付物的一部分，缺少任一项都必须使角色导出失败。

#### 场景:缺少材质或技能复刻关键输入
- **当** 某个角色的模型、动画、PNG 纹理、材质清单、技能配置或参数关联摘要中的任一项缺失、生成失败或无法与角色摘要对齐
- **那么** 系统必须将该角色导出标记为失败，并在摘要中指出缺失项；禁止将该角色视为"部分成功但可导入"

### 需求:逐动画 root-motion 必须纳入正式摘要
SekiroExporter 必须把每段动画的 root-motion 结果纳入角色级正式摘要，禁止把 root-motion 正确性留给 UE 端猜测。

#### 场景:导出动画 root-motion 摘要
- **当** 系统导出某个动画 clip
- **那么** 正式摘要必须记录该 clip 的 root-motion 是否成功导出、来源骨骼或来源数据、时间基准、位移轴约定和失败原因；若该 clip 需要 root-motion 但未成功交付，角色导出必须失败

### 需求:正式资产包与调试产物边界
SekiroExporter 必须明确区分正式资产包与调试产物，正式成功判定只依赖规范交付物。

#### 场景:调试产物不影响正式成功判定
- **当** 用户使用 `--keep-intermediates` 或其他调试选项导出
- **那么** 系统必须将额外生成的中间文件、临时格式文件、调试日志等标记为非正式产物；这些文件不得进入正式资产包摘要，也不得影响角色级正式成功判定

#### 场景:非正式格式不得视为正式交付物
- **当** 系统生成 DAE、FBX、DDS 或其他中间/调试格式文件
- **那么** 这些文件不得计入正式资产包摘要中的交付物列表；只有 glTF 2.0 模型、glTF 2.0 动画、PNG 纹理和正式 JSON 清单才可被记为正式交付物

#### 场景:正式资产包摘要只包含规范交付物
- **当** 系统输出角色级正式资产包摘要
- **那么** 摘要必须只包含模型 glTF、动画 glTF、PNG 纹理、材质清单 JSON、技能配置 JSON、参数摘要和 root-motion 元数据；任何非正式产物必须列在单独的 `debugArtifacts` 或 `unexpectedArtifacts` 字段中

### 需求:角色级动画完整性必须基于期望 clip 集合判定
SekiroExporter 必须先确定角色正式应交付的动画 clip 集合，再按该集合作为成功条件；禁止把"导出到至少一个动画文件"视为角色动画成功。

#### 场景:主角 supplemental ANIBND 动画必须纳入正式集合
- **当** 某个角色（例如 `c0000`）的正式动画解析需要 base ANIBND 之外的 supplemental player ANIBND、共享动作池或延迟加载 ANIBND
- **那么** 系统必须把这些来源解析出的正式 clip 一并纳入角色级期望动画集合、逐 clip 写入正式摘要，并在缺少任一应交付 clip 时将角色导出标记为失败

#### 场景:导出结果缺少任一应交付 clip
- **当** 系统已经为某个角色解析出期望动画集合，但最终导出目录、正式摘要或动画分辨结果中缺少任一 clip
- **那么** 系统必须将该角色动画导出标记为失败，并在角色级摘要中记录期望 clip、缺失 clip、来源 ANIBND 与失败原因

### 需求:期望动画集合必须显式定义并记录
SekiroExporter 必须在导出开始前显式定义每个角色的期望动画 clip 集合，并在摘要中记录。

#### 场景:定义 c0000 主角期望动画集合
- **当** 系统导出 `c0000` 主角
- **那么** 系统必须先解析 base ANIBND、supplemental player ANIBND（含钩锁动作）和共享动作池，确定完整的期望 clip 列表，并在角色级摘要中记录期望动画数量、来源 ANIBND 路径和每个 clip 的正式标识

#### 场景:期望动画集合解析失败时角色导出失败
- **当** 系统无法解析某个角色的期望动画集合（例如 supplemental ANIBND 缺失或损坏）
- **那么** 系统必须将该角色导出标记为失败，并在摘要中记录无法确定期望动画集合的原因

#### 场景:禁止单独动画数量大于零视为成功
- **当** 系统评估角色动画导出是否成功
- **那么** 系统必须检查期望 clip 集合是否完整导出，而非仅检查导出目录中是否存在至少一个动画文件；缺少任一期望 clip 必须导致角色动画导出失败

### 需求:角色级摘要必须包含完整动画统计信息
SekiroExporter 必须在角色级摘要中输出完整的动画导出统计，供 UE 导入和验收直接消费。

#### 场景:输出期望动画数量和成功导出数量
- **当** 系统完成角色动画导出
- **那么** 角色级摘要必须包含 `expectedAnimationCount`、`exportedAnimationCount` 和 `missingAnimationFiles` 字段

#### 场景:输出缺失动画标识和来源 ANIBND
- **当** 某些期望 clip 未成功导出
- **那么** 系统必须在摘要中记录每个缺失 clip 的正式标识（如 `a000_202010.gltf`）和其来源 ANIBND 路径

#### 场景:输出 root-motion 完整性统计
- **当** 系统完成角色动画导出
- **那么** 角色级摘要必须包含 `missingRootMotionFiles` 列表，记录哪些 clip 缺少正式 root-motion 数据
