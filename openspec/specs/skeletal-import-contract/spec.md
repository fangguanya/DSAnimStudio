## ADDED Requirements

### 需求:UE 正式输入必须与单一导出合同一致
UE 侧正式导入链路必须以 glTF 2.0 模型、glTF 2.0 动画和 PNG 纹理作为唯一正式输入，并与正式导出合同保持一致。

#### 场景:导入正式角色资产
- **当** UE 导入器处理正式导出物
- **那么** 它必须按 glTF 2.0 和 PNG 合同完成资产创建与验证

### 需求:UE 导入必须验证 formal skeleton root 与动画绑定
UE 侧正式导入验证必须确认模型存在唯一 formal skeleton root，动画 clip 与该 skeleton 可绑定，且不会依赖历史兼容路径或导入期猜测修补。

#### 场景:导入正式模型与单 clip 动画
- **当** UE 同时处理某角色的正式模型和正式动画交付物
- **那么** 它必须能形成可绑定的 Skeleton/Animation 关系，否则该角色不得计为正式导入成功

## MODIFIED Requirements

### 需求:不符合正式合同的输入不得计为导入成功
如果输入资产缺失正式骨架关系、缺失正式动画绑定、缺失正式纹理交付，或者依赖 FBX、DDS、Collada、宽松兼容字段才能导入，则该输入不得被计为正式导入成功。

#### 场景:输入依赖历史兼容路径
- **当** 一个角色资产只能通过历史兼容路径而非正式合同导入 UE
- **那么** 导入验证必须失败并在验收报告中说明原因
