## ADDED Requirements

### 需求:正式纹理导出不得回退为 DDS
纹理导出能力在正式模式下必须只交付 PNG。正式模式下禁止以 DDS 转存替代 PNG 成功结果。

#### 场景:PNG 解码失败
- **当** 纹理无法被正式解码为 PNG
- **那么** 该纹理必须被记为失败，而不是回退为 DDS 文件

## MODIFIED Requirements

### 需求:纹理导出成功必须基于正式支持矩阵
纹理导出必须以正式支持矩阵为准决定成功与失败。对未支持的源编码必须明确报错，并反映到角色级验收，而不是仅输出 warning 后继续统计为成功。

#### 场景:角色含未支持纹理
- **当** 角色纹理中存在未纳入正式支持矩阵的编码格式
- **那么** 该角色的纹理导出状态必须为失败
