## ADDED Requirements

### 需求:技能配置必须由模板和正式参数定义驱动
正式技能配置导出必须由正式模板与正式参数定义驱动，输出字段名、数据类型、枚举和值域必须来源于可追溯定义。正式链路禁止在缺模板或缺定义时直接退化为 raw bytes。

#### 场景:导出技能配置
- **当** 导出器正式导出角色技能配置
- **那么** 每个事件参数都必须以正式字段名和正式数据类型输出，且数据来源可追溯

### 需求:参数表读取必须移除 hack 解析器
正式链路读取 Sekiro 参数表时必须使用正式定义驱动的解析方式，禁止继续将 `PARAM_Hack` 作为正式交付所依赖的读取路径。

#### 场景:读取 EquipParam 或相关参数表
- **当** 导出器读取正式参数表以生成角色装配、技能或行为配置
- **那么** 读取结果必须来自正式解析器，而不是 hack 读取器

### 需求:未知字段不得猜测性解析
当模板字段、参数字段或类型定义缺失时，正式链路必须显式失败或报告未完成映射，禁止将未知类型按 `s32` 猜读或以占位名掩盖缺口。

#### 场景:模板存在未知字段类型
- **当** 技能事件模板包含当前版本未建模的数据类型
- **那么** 导出必须报告该字段未正式支持，而不是自动改按其他类型读取

### 需求:技能配置必须记录 TAE 与交付动画的求解关系
正式技能配置 schema 必须能区分原始 TAE 条目、求解后的 HKX 动画和最终交付动画文件之间的关系，确保技能配置可追溯到正式解析链路。

#### 场景:导出带引用链的 Sekiro 技能
- **当** 技能条目的 TAE 标识、HKX 标识和交付动画文件名不相同
- **那么** 正式输出必须保留这些关系所需的元数据，而不是仅保留一个模糊的动画名字段

### 需求:模板有效载荷后的额外字节只能作为显式零填充处理
正式技能参数解析必须先按模板定义读取有效载荷；若事件参数块包含额外尾字节，这些字节只能在全部为零时被视为保留填充，否则必须显式失败。

#### 场景:TAE 事件参数长度大于模板有效载荷长度
- **当** 一个事件参数块比当前模板定义更长
- **那么** 导出器只能接受全零尾字节，并且不得把这些尾字节提升为未定义正式字段
