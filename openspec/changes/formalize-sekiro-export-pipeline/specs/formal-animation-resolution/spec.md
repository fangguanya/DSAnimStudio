## ADDED Requirements

### 需求:正式动画求解必须使用单一共享解析函数
正式链路必须通过单一共享解析函数求解动画请求，统一服务于动画导出、技能导出、验收报告和调试输出。禁止不同调用方各自实现一套 TAE/HKX/文件名 推断逻辑。

#### 场景:动画导出与技能导出同时请求同一动作
- **当** 动画导出和技能配置导出都需要解析同一个 Sekiro 动作请求
- **那么** 两者必须得到同一份求解结果，并共享同一套 request id、resolved id、HKX id、source anim file 和 deliverable anim file 字段

### 需求:正式动画求解必须区分请求标识与交付标识
正式链路必须显式区分动画请求入口和最终交付对象，至少包含请求的 TAE 条目 ID、引用链求解后的 TAE 条目 ID、求解后的 HKX ID、`SourceAnimFileName` 和最终交付动画文件名。禁止把这些字段压缩为单一 anim id。

#### 场景:请求条目和实际动画不一致
- **当** 一个 TAE 条目通过引用链或源动画文件映射到另一个 HKX 动画
- **那么** 正式求解结果必须同时保留请求入口和最终交付对象，而不是覆盖原始请求字段

### 需求:动画源与技能源必须可独立声明
正式链路必须允许动画导出和技能导出来自不同的 ANIBND/TAE 源，并在求解结果中记录各自来源。禁止默认假定“播放用动画源”和“技能事件源”一定来自同一个容器。

#### 场景:主角动作使用独立技能源
- **当** 角色的动画采样来自一个 ANIBND，而技能事件来自另一个 ANIBND 或另一组 TAE 文件
- **那么** 正式求解结果和验收报告必须分别记录 animation source 与 skill source

### 需求:求解失败或歧义必须显式失败
当请求的 TAE 条目不存在、引用链无法闭合、HKX ID 无法求解、`SourceAnimFileName` 无法映射到正式交付动画，或多个候选结果无法通过正式规则消歧时，正式链路必须显式失败。禁止按名称模糊匹配、首个命中或默认相同 ID 继续导出。

#### 场景:同名动画存在多个候选
- **当** 当前角色的 TAE/ANIBND 中存在多个可能对应请求的动画候选
- **那么** 正式导出必须按预定义规则完成唯一求解，否则该动画必须失败并记录歧义原因

### 需求:正式交付与报告必须复用求解元数据
正式动画交付文件命名、技能配置动画元数据和角色验收报告必须复用同一份动画求解元数据，确保 CLI、导出物和验收报告可以互相校验。

#### 场景:角色导出完成
- **当** 正式链路完成一个角色的模型、动画和技能导出
- **那么** 动画文件名、技能配置中的动画元数据和验收报告中的动画解析字段必须彼此一致
