## ADDED Requirements

### 需求:Sekiro HKX 必须使用唯一正式解析路径
正式导出链路对 Sekiro 骨架与动画 HKX 的读取必须使用单一、可声明的正式解析策略。正式链路禁止通过 `HKXVariation` 轮询、tagfile/legacy 混合兜底或“哪个能读就用哪个”的方式决定解析结果。

#### 场景:解析正式动画 HKX
- **当** 导出器读取 Sekiro 骨架或动画 HKX
- **那么** 它必须走已声明的唯一正式解析路径，并在日志与验收报告中可追溯

### 需求:动画采样失败必须显式失败
正式动画构建时，任意骨骼在任意帧无法得到合法变换，都必须导致该动画导出失败。正式链路禁止以 identity 平移、identity 旋转或单位缩放填补缺失帧。

#### 场景:某骨骼某帧采样异常
- **当** 动画采样过程中某一骨骼帧变换无法读取或无法分解
- **那么** 当前动画 clip 必须被标记为失败，并写入失败原因

### 需求:骨架与动画解析结果必须保持一致
正式链路必须保证骨架节点集、动画节点集、skin joints 和 inverse bind matrices 使用同一套骨骼标识和层级语义，禁止通过后续猜测匹配来补齐不一致。

#### 场景:动画导出完成
- **当** 一个动画 clip 被正式导出为 glTF
- **那么** clip 中被引用的节点必须全部能映射到正式骨架节点集，且该关系无需依赖名称猜测或补丁修正
