## 新增需求

### 需求:正式纹理输出与材质清单一致
纹理导出必须为材质清单中每个正式引用提供唯一且稳定的纹理交付物，使 UE 侧能够按清单直接绑定。

#### 场景:输出可被材质清单唯一引用的 PNG
- **当** 某个角色执行正式纹理导出
- **那么** 每个被材质清单正式引用的纹理都必须生成唯一的 PNG 文件和稳定相对路径，并与材质清单中的绑定信息一一对应

### 需求:正式引用纹理缺失时失败
纹理导出不得把材质重建所需纹理的缺失视为可接受状态。

#### 场景:材质清单引用纹理无法导出
- **当** 某个被材质清单正式引用的纹理无法解码、命名冲突无法消解或未生成最终 PNG 文件
- **那么** 系统必须将该角色的纹理导出标记为失败，并将失败信息反馈给角色级正式资产包摘要

### 需求:纹理编码支持矩阵
系统必须建立 Sekiro 当前使用纹理编码的正式支持矩阵，并按支持矩阵判定成功/失败。

#### 场景:支持的纹理格式导出成功
- **当** 纹理使用 BC7、DXT1、DXT3、DXT5、BC4、BC5、BC6H、ATI1、ATI2、A8 或 R8G8B8A8 格式
- **那么** 系统必须成功将该纹理转换为 PNG 格式

#### 场景:不支持的纹理格式导出失败
- **当** 纹理使用不支持解码的格式
- **那么** 系统必须将该纹理导出标记为失败，并记录不支持的格式类型

### 需求:正式 PNG 输出命名和色彩空间契约
系统必须为正式 PNG 输出定义统一命名、色彩/通道语义和材质引用契约。

#### 场景:纹理命名稳定且可预测
- **当** 系统导出纹理
- **那么** 输出文件名必须基于原始纹理名，使用小写字母、数字和下划线，并以 `.png` 为扩展名

#### 场景:色彩空间根据纹理类型确定
- **当** 系统导出 BaseColor/Diffuse/Emissive 类型纹理
- **那么** 必须标记为 sRGB 色彩空间
- **当** 系统导出 Normal/Specular/Roughness 类型纹理
- **那么** 必须标记为 Linear 色彩空间
