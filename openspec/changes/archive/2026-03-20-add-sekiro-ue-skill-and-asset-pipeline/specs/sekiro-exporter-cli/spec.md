## 新增需求

### 需求:角色级正式复刻资产包摘要
SekiroExporter 必须为每个角色输出一份可供 UE 复刻链路消费的正式资产包摘要，明确记录资产完整性和关联关系。

#### 场景:导出角色正式资产包摘要
- **当** 用户执行单角色或批量角色导出
- **那么** 系统必须为每个角色输出正式摘要，至少记录模型交付物、动画交付物、PNG 纹理、材质清单、技能配置、参数关联摘要、root-motion 完整性以及这些文件的稳定相对路径和完成状态

### 需求:复刻输入缺失时角色导出失败
SekiroExporter 必须把 UE 复刻所需的关键输入视为正式交付物的一部分，缺少任一项都必须使角色导出失败。

#### 场景:缺少材质或技能复刻关键输入
- **当** 某个角色的模型、动画、PNG 纹理、材质清单、技能配置或参数关联摘要中的任一项缺失、生成失败或无法与角色摘要对齐
- **那么** 系统必须将该角色导出标记为失败，并在摘要中指出缺失项；禁止将该角色视为“部分成功但可导入”

### 需求:逐动画 root-motion 必须纳入正式摘要
SekiroExporter 必须把每段动画的 root-motion 结果纳入角色级正式摘要，禁止把 root-motion 正确性留给 UE 端猜测。

#### 场景:导出动画 root-motion 摘要
- **当** 系统导出某个动画 clip
- **那么** 正式摘要必须记录该 clip 的 root-motion 是否成功导出、来源骨骼或来源数据、时间基准、位移轴约定和失败原因；若该 clip 需要 root-motion 但未成功交付，角色导出必须失败

### 需求:正式资产包与调试产物边界
SekiroExporter 必须明确区分正式资产包与调试产物，正式成功判定只依赖规范交付物。

#### 场景:调试产物不影响正式成功判定
- **当** 用户使用 `--keep-intermediates` 或其他调试选项导出
- **那么** 系统必须将额外生成的中间文件、临时格式文件、调试日志等标记为非正式产物；这些文件不得进入正式资产包摘要，也不得影响角色级正式成功判定

#### 场景:非正式格式不得视为正式交付物
- **当** 系统生成 DAE、FBX、DDS 或其他中间/调试格式文件
- **那么** 这些文件不得计入正式资产包摘要中的交付物列表；只有 glTF 2.0 模型、glTF 2.0 动画、PNG 纹理和正式 JSON 清单才可被记为正式交付物

#### 场景:正式资产包摘要只包含规范交付物
- **当** 系统输出角色级正式资产包摘要
- **那么** 摘要必须只包含模型 glTF、动画 glTF、PNG 纹理、材质清单 JSON、技能配置 JSON、参数摘要和 root-motion 元数据；任何非正式产物必须列在单独的 `debugArtifacts` 或 `unexpectedArtifacts` 字段中

### 需求:角色级动画完整性必须基于期望 clip 集合判定
SekiroExporter 必须先确定角色正式应交付的动画 clip 集合，再按该集合作为成功条件；禁止把“导出到至少一个动画文件”视为角色动画成功。

#### 场景:主角 supplemental ANIBND 动画必须纳入正式集合
- **当** 某个角色（例如 `c0000`）的正式动画解析需要 base ANIBND 之外的 supplemental player ANIBND、共享动作池或延迟加载 ANIBND
- **那么** 系统必须把这些来源解析出的正式 clip 一并纳入角色级期望动画集合、逐 clip 写入正式摘要，并在缺少任一应交付 clip 时将角色导出标记为失败

#### 场景:导出结果缺少任一应交付 clip
- **当** 系统已经为某个角色解析出期望动画集合，但最终导出目录、正式摘要或动画分辨结果中缺少任一 clip
- **那么** 系统必须将该角色动画导出标记为失败，并在角色级摘要中记录期望 clip、缺失 clip、来源 ANIBND 与失败原因

### 需求:期望动画集合必须显式定义并记录
SekiroExporter 必须在导出开始前显式定义每个角色的期望动画 clip 集合，并在摘要中记录。

#### 场景:定义 c0000 主角期望动画集合
- **当** 系统导出 `c0000` 主角
- **那么** 系统必须先解析 base ANIBND、supplemental player ANIBND（含钩锁动作）和共享动作池，确定完整的期望 clip 列表，并在角色级摘要中记录期望动画数量、来源 ANIBND 路径和每个 clip 的正式标识

#### 场景:期望动画集合解析失败时角色导出失败
- **当** 系统无法解析某个角色的期望动画集合（例如 supplemental ANIBND 缺失或损坏）
- **那么** 系统必须将该角色导出标记为失败，并在摘要中记录无法确定期望动画集合的原因

#### 场景:禁止单独动画数量大于零视为成功
- **当** 系统评估角色动画导出是否成功
- **那么** 系统必须检查期望 clip 集合是否完整导出，而非仅检查导出目录中是否存在至少一个动画文件；缺少任一期望 clip 必须导致角色动画导出失败

### 需求:角色级摘要必须包含完整动画统计信息
SekiroExporter 必须在角色级摘要中输出完整的动画导出统计，供 UE 导入和验收直接消费。

#### 场景:输出期望动画数量和成功导出数量
- **当** 系统完成角色动画导出
- **那么** 角色级摘要必须包含 `expectedAnimationCount`、`exportedAnimationCount` 和 `missingAnimationFiles` 字段

#### 场景:输出缺失动画标识和来源 ANIBND
- **当** 某些期望 clip 未成功导出
- **那么** 系统必须在摘要中记录每个缺失 clip 的正式标识（如 `a000_202010.gltf`）和其来源 ANIBND 路径

#### 场景:输出 root-motion 完整性统计
- **当** 系统完成角色动画导出
- **那么** 角色级摘要必须包含 `missingRootMotionFiles` 列表，记录哪些 clip 缺少正式 root-motion 数据
