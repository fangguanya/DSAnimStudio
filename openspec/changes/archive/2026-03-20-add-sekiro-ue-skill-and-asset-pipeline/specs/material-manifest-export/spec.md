## 新增需求

### 需求:UE 材质实例重建清单
材质清单导出必须包含足以在 UE 中稳定重建材质实例的正式字段，禁止只提供模糊的纹理槽位映射。

#### 场景:导出材质实例重建键
- **当** 系统导出某个角色的材质清单
- **那么** 每个材质条目必须包含稳定的材质实例键、角色内材质槽标识、所需纹理参数键以及与导出纹理文件一一对应的绑定信息，使 UE 端可以不依赖文件名猜测直接创建材质实例

### 需求:材质清单与纹理导出严格对齐
材质清单导出必须对纹理引用的完整性负责，缺少正式纹理输入时不得继续生成看似完整的清单。

#### 场景:引用纹理未形成正式交付物
- **当** 某个材质条目引用的纹理未能导出为正式文件，或无法映射到唯一的 UE 材质参数键
- **那么** 材质清单导出必须失败并指出具体材质和纹理引用；禁止省略该引用或用占位值继续导出

### 需求:材质清单必须固定正式父材质合同
材质清单导出必须显式服务唯一正式 Master Material 合同，禁止让 UE 导入侧自行决定父材质模板。

#### 场景:导出正式父材质参数合同
- **当** 系统导出某个角色的材质清单
- **那么** 每个正式材质条目都必须只使用与唯一正式 Master Material 兼容的参数键集合和绑定语义；禁止输出只有在 `DefaultMaterial`、测试父材质或命名猜测 fallback 下才成立的临时槽位定义

### 需求:UE 端必须使用唯一正式 Master Material
UE 导入链路必须将所有正式材质实例绑定到唯一正式 Master Material，禁止使用默认材质或临时父材质。

#### 场景:材质实例必须引用正式 Master Material
- **当** UE 导入脚本创建或更新角色材质实例
- **那么** 每个材质实例的父材质必须是仓库约定的唯一正式 Master Material（例如 `M_SekiroCharacter`），而不是 `DefaultMaterial`、引擎默认材质或临时创建的测试材质

#### 场景:禁止 DefaultMaterial 进入正式路径
- **当** UE 导入脚本无法找到正式 Master Material 或 manifest 参数绑定失败
- **那么** 系统必须将该角色的材质导入标记为失败，而不是使用 `DefaultMaterial` 作为 fallback 父材质

#### 场景:禁止命名猜测槽位
- **当** UE 导入脚本处理材质纹理绑定
- **那么** 系统必须只使用 manifest 中显式声明的参数键；禁止根据文件名后缀（如 `_n`、`_s`）或其他命名约定自动猜测材质参数槽位
