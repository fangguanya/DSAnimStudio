## 新增需求

### 需求:角色复刻会话必须消费正式导入资产
SekiroSkillEditor 插件必须基于正式导入后的 UE 资产建立角色复刻会话，禁止在编辑器内重新猜测或临时拼装资源。

#### 场景:打开角色复刻会话
- **当** 用户在 `SekiroSkillEditor` 中打开某个角色或技能
- **那么** 插件必须加载该角色的骨骼网格、材质实例、技能数据资产、参数缓存和角色数据资产中的正式引用，并据此构建编辑器会话；若任一正式引用缺失，插件必须将该角色标记为不可验收

### 需求:插件必须提供自定义 Sekiro 时间轴编辑器
SekiroSkillEditor 插件必须在 Unreal 内提供一个 Sekiro 专用的自定义时间轴编辑器，而不是只读浏览器或简化列表视图。

#### 场景:技能资产以资产编辑器方式打开
- **当** 用户在 Content Browser 中双击某个正式导入的技能资产
- **那么** 插件必须打开该技能对应的 Sekiro 资产编辑器会话，加载该技能的正式动画、角色、材质、参数和时间轴数据；禁止要求用户先打开一个全局 Nomad Tab 再从浏览树中手动选择技能，才能进入正式编辑流程

#### 场景:打开技能时间轴编辑器
- **当** 用户在 `SekiroSkillEditor` 中打开某个角色或技能
- **那么** 插件必须显示一个可编辑的 Sekiro 时间轴会话，至少覆盖动画/事件列表、分轨时间轴、区间事件、播放头、属性面板和角色预览联动；这些视图必须共同消费同一份正式技能数据，而不是各自维护临时状态

#### 场景:`skill_config` 作为正式序列化来源
- **当** 用户在自定义时间轴编辑器中查看、修改或验证某个技能事件
- **那么** 编辑器必须以 `skill_config` 对应的数据结构作为正式来源和回写目标；禁止让 UI 状态、临时缓存或仅编辑器内存在的数据字段成为另一套事实来源

#### 场景:编辑器修改必须形成正式回写
- **当** 用户在技能资产编辑器中修改事件顺序、轨道、参数字段、语义链接或与 root-motion 对齐相关的正式数据
- **那么** 插件必须提供与 `skill_config` 严格等价的正式回写路径，并让保存动作更新该正式来源；禁止仅修改内存态、临时 Slate 状态或仅 Unreal 资产内存在的派生字段

### 需求:技能语义复刻必须展示解析后的实际结果
SekiroSkillEditor 插件必须展示 Sekiro 技能事件解析后的实际语义结果，而不只是原始事件数据。

#### 场景:显示攻击与特效语义结果
- **当** 时间轴播放头命中攻击、特效、义手忍具覆盖或战技样式检查相关事件
- **那么** 3D 视口和属性面板必须显示解析后的参数链路、实际生效的 DummyPoly、覆盖后的结果、关联的 `BehaviorParam`、`AtkParam`、`SpEffectParam` 或样式检查结果，以及该时刻生效的 root-motion 结果

### 需求:编辑器会话必须可检查 root-motion
SekiroSkillEditor 插件必须在时间轴和角色预览中提供 root-motion 检查能力，确保位移语义属于正式复刻结果的一部分。

#### 场景:检查动画 root-motion
- **当** 用户在时间轴编辑器中播放、拖动或逐帧检查某个技能动画
- **那么** 编辑器必须能够显示并验证该动画的正式 root-motion 结果，包括时间轴对齐、位移方向和预览中的实际位移；若 root-motion 缺失、错轴或与正式导出摘要不一致，角色必须标记为不可验收

### 需求:编辑器内角色级复刻验证
SekiroSkillEditor 插件必须提供角色级验证入口，以证明当前角色在编辑器内已达到正式复刻标准。

#### 场景:执行角色级复刻验证
- **当** 用户在插件中执行某个角色的复刻验证
- **那么** 系统必须检查正式资产引用、材质实例绑定、动画绑定、技能资产、参数索引、自定义时间轴编辑器会话和关键技能语义/root-motion 复刻结果；任一项失败都必须使该角色验证失败，禁止降级为仅警告

#### 场景:正式材质或动画完整性不满足时拒绝验收
- **当** 当前技能会话所依赖的角色材质实例未绑定到唯一正式 Master Material，或该角色缺少正式摘要中要求的任一动画 clip
- **那么** 插件必须将该角色会话标记为不可验收，并在编辑器验证结果中明确指出具体缺失的材质装配项或动画 clip
