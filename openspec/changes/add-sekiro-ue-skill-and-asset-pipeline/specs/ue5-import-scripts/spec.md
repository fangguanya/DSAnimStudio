## 新增需求

### 需求:角色级资源编排导入
UE5 导入脚本必须把 Sekiro 正式资产包导入为彼此正确关联的 UE 资产集合，而不只是独立文件。

#### 场景:导入角色正式资产包
- **当** 用户对某个角色运行正式导入脚本并提供模型、动画、纹理、材质清单、技能配置和参数关联摘要
- **那么** 脚本必须创建或更新该角色的 `USkeletalMesh`、`USkeleton`、纹理资产、材质实例、技能数据资产、参数缓存资产和角色数据资产，并建立它们之间的正式引用关系

### 需求:材质与技能导入必须由正式清单驱动
UE5 导入脚本必须以正式清单和正式摘要为唯一依据建立资源关系，禁止按文件名约定或目录猜测进行兜底。

#### 场景:按正式清单建立材质和技能引用
- **当** 导入脚本处理材质清单和技能配置
- **那么** 脚本必须按正式清单中的材质实例键、纹理参数键、动画文件引用和参数索引键建立资源关系；禁止因为文件名相似或目录中存在候选资源就自动绑定

### 需求:角色级导入失败必须阻断成功结论
UE5 导入脚本必须把资源关系建立失败视为角色级失败，而不是容忍不完整导入。

#### 场景:资源关系未建立完整
- **当** 某个角色的 SkeletalMesh、材质实例、动画绑定、技能资产或参数缓存中的任一关系未建立成功
- **那么** 脚本必须将该角色标记为失败，并输出缺失的资源关系；批量导入不得将该角色计入成功总数
