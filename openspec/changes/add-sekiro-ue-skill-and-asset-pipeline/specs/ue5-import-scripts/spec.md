## 新增需求

### 需求:角色级资源编排导入
UE5 导入脚本必须把 Sekiro 正式资产包导入为彼此正确关联的 UE 资产集合，而不只是独立文件。

#### 场景:导入角色正式资产包
- **当** 用户对某个角色运行正式导入脚本并提供模型、动画、纹理、材质清单、技能配置和参数关联摘要
- **那么** 脚本必须创建或更新该角色的 `USkeletalMesh`、`USkeleton`、纹理资产、材质实例、技能数据资产、参数缓存资产和角色数据资产，并建立它们之间的正式引用关系与 root-motion 绑定结果

### 需求:材质与技能导入必须由正式清单驱动
UE5 导入脚本必须以正式清单和正式摘要为唯一依据建立资源关系，禁止按文件名约定或目录猜测进行兜底。

#### 场景:按正式清单建立材质和技能引用
- **当** 导入脚本处理材质清单和技能配置
- **那么** 脚本必须按正式清单中的材质实例键、纹理参数键、动画文件引用和参数索引键建立资源关系；禁止因为文件名相似或目录中存在候选资源就自动绑定

### 需求:角色级导入失败必须阻断成功结论
UE5 导入脚本必须把资源关系建立失败视为角色级失败，而不是容忍不完整导入。

#### 场景:资源关系未建立完整
- **当** 某个角色的 SkeletalMesh、材质实例、动画绑定、技能资产或参数缓存中的任一关系未建立成功
- **那么** 脚本必须将该角色标记为失败，并输出缺失的资源关系；批量导入不得将该角色计入成功总数

### 需求:导入链路必须消费正式 root-motion
UE5 导入脚本必须按正式摘要消费动画 root-motion，并把结果绑定到 UE 动画与角色会话中。

#### 场景:导入动画 root-motion
- **当** 导入脚本处理某个包含正式 root-motion 的动画 clip
- **那么** 脚本必须根据正式摘要建立对应的 root-motion 绑定或配置，并在角色级导入报告中记录该 clip 的 root-motion 是否成功消费；禁止默默忽略或在未记录的情况下改用 UE 默认提取逻辑

### 需求:UE 材质实例必须使用唯一正式 Master Material
UE5 导入脚本必须把正式 manifest 重建的材质实例绑定到唯一正式 Master Material，禁止继续以默认父材质或临时父材质报告成功。

#### 场景:使用正式 Master Material 创建材质实例
- **当** 导入脚本根据正式材质清单创建或更新角色材质实例
- **那么** 每个正式材质实例都必须绑定到仓库约定的唯一正式 Master Material，并按 manifest 中的正式参数键设置纹理和标量参数；禁止以 `DefaultMaterial`、测试材质或多套并存的父材质模板进入正式路径

#### 场景:manifest 参数绑定失败时阻断成功
- **当** 某个正式材质实例缺少父材质、缺少 manifest 声明的参数绑定，或只能依赖命名后缀/目录扫描猜测补槽位
- **那么** 导入脚本必须将该角色标记为失败，并在导入报告中记录具体材质实例和缺失参数；禁止保留自动猜测兜底后仍报告成功

### 需求:角色级动画导入必须校验完整覆盖率
UE5 导入脚本必须按正式摘要中的期望动画集合检查导入结果，禁止仅以“成功导入了一部分动画”得出角色成功结论。

#### 场景:期望动画集合存在缺口
- **当** 正式摘要列出了某个角色的期望动画 clip 集合，但 UE 导入后缺少任一 clip、Skeleton 绑定错误或动画资产无法解析
- **那么** 导入脚本必须将该角色标记为失败，并在导入报告中记录缺失或错误绑定的 clip；禁止把共享动作池、supplemental ANIBND 或钩锁相关动画降级为仅警告
