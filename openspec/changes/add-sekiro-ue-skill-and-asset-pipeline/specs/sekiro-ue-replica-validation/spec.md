## 新增需求

### 需求:角色级完整复刻验收
系统必须以角色为单位验证 Sekiro 正式导出资产在 UE 中是否被完整复刻，禁止以“存在部分资产”替代正式成功。

#### 场景:角色完整复刻成功
- **当** 验证器检查某个角色的正式资产包、UE 导入结果和 `SekiroSkillEditor` 复刻结果
- **那么** 只有在模型、动画、PNG 纹理、材质实例、技能数据资产、参数索引、自定义时间轴编辑器会话以及关键技能语义/root-motion 复刻结果全部成立时，系统才必须将该角色标记为成功

#### 场景:任一关键链路缺失时失败
- **当** 某个角色缺少正式资产包中的任一关键交付物，或 UE 端未建立 SkeletalMesh、材质实例、技能资产、参数索引、DummyPoly 语义复刻中的任一项
- **那么** 验证器必须立即将该角色标记为失败，并记录缺失链路；系统禁止把该结果降级为警告或部分成功

### 需求:root-motion 必须纳入角色级验收
系统必须把 root-motion 正确性视为 Sekiro 角色复刻验收的一部分，而不是可选增强项。

#### 场景:验证 root-motion 结果
- **当** 验证器检查某个角色的动画复刻结果
- **那么** 系统必须验证正式摘要、UE 导入结果和编辑器预览中的 root-motion 是否一致；只要存在缺失、错轴、静止或时间基准不一致的 root-motion，角色验收就必须失败

### 需求:批量复刻验收报告
系统必须输出可追溯的批量角色验收报告，逐个角色说明成功或失败原因。

#### 场景:批量执行角色级验收
- **当** 用户对多个 Sekiro 角色执行完整复刻验收
- **那么** 系统必须输出逐角色结果清单，至少包含角色 ID、失败阶段、失败原因和是否满足正式复刻合同；只要存在失败角色，批量结果就必须为失败

### 需求:正式材质路径必须纳入角色级验收
系统必须把角色正式材质装配是否使用唯一 Master Material 和完整 manifest 参数绑定视为角色级复刻验收的一部分。

#### 场景:角色材质实例仍停留在临时父材质路径
- **当** 某个角色的材质实例仍依赖 `DefaultMaterial`、临时父材质、自动猜测槽位或其他非正式材质路径
- **那么** 验证器必须将该角色标记为失败，并记录具体材质实例和缺失的正式装配项

### 需求:角色级动画覆盖率必须纳入验收
系统必须验证角色在正式摘要中声明的期望动画集合是否已经完整导出、导入并可在编辑器会话中绑定使用。

#### 场景:角色缺少期望动画 clip
- **当** 某个角色的正式摘要、UE 导入结果或编辑器会话缺少任一应交付动画 clip，包括 supplemental player ANIBND、共享动作池或钩锁相关动作
- **那么** 验证器必须将该角色标记为失败，并输出缺失 clip、来源 ANIBND 与失败阶段；禁止把该结果降级为 warning 或“共享池动画可人工忽略”
