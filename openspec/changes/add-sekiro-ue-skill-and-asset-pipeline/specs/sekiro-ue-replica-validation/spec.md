## 新增需求

### 需求:角色级完整复刻验收
系统必须以角色为单位验证 Sekiro 正式导出资产在 UE 中是否被完整复刻，禁止以“存在部分资产”替代正式成功。

#### 场景:角色完整复刻成功
- **当** 验证器检查某个角色的正式资产包、UE 导入结果和 `SekiroSkillEditor` 复刻结果
- **那么** 只有在模型、动画、PNG 纹理、材质实例、技能数据资产、参数索引以及关键技能语义复刻结果全部成立时，系统才必须将该角色标记为成功

#### 场景:任一关键链路缺失时失败
- **当** 某个角色缺少正式资产包中的任一关键交付物，或 UE 端未建立 SkeletalMesh、材质实例、技能资产、参数索引、DummyPoly 语义复刻中的任一项
- **那么** 验证器必须立即将该角色标记为失败，并记录缺失链路；系统禁止把该结果降级为警告或部分成功

### 需求:批量复刻验收报告
系统必须输出可追溯的批量角色验收报告，逐个角色说明成功或失败原因。

#### 场景:批量执行角色级验收
- **当** 用户对多个 Sekiro 角色执行完整复刻验收
- **那么** 系统必须输出逐角色结果清单，至少包含角色 ID、失败阶段、失败原因和是否满足正式复刻合同；只要存在失败角色，批量结果就必须为失败
