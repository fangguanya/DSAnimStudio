## 新增需求

### 需求:角色复刻会话必须消费正式导入资产
SekiroSkillEditor 插件必须基于正式导入后的 UE 资产建立角色复刻会话，禁止在编辑器内重新猜测或临时拼装资源。

#### 场景:打开角色复刻会话
- **当** 用户在 `SekiroSkillEditor` 中打开某个角色或技能
- **那么** 插件必须加载该角色的骨骼网格、材质实例、技能数据资产、参数缓存和角色数据资产中的正式引用，并据此构建编辑器会话；若任一正式引用缺失，插件必须将该角色标记为不可验收

### 需求:技能语义复刻必须展示解析后的实际结果
SekiroSkillEditor 插件必须展示 Sekiro 技能事件解析后的实际语义结果，而不只是原始事件数据。

#### 场景:显示攻击与特效语义结果
- **当** 时间轴播放头命中攻击、特效、义手忍具覆盖或战技样式检查相关事件
- **那么** 3D 视口和属性面板必须显示解析后的参数链路、实际生效的 DummyPoly、覆盖后的结果以及关联的 `BehaviorParam`、`AtkParam`、`SpEffectParam` 或样式检查结果

### 需求:编辑器内角色级复刻验证
SekiroSkillEditor 插件必须提供角色级验证入口，以证明当前角色在编辑器内已达到正式复刻标准。

#### 场景:执行角色级复刻验证
- **当** 用户在插件中执行某个角色的复刻验证
- **那么** 系统必须检查正式资产引用、材质实例绑定、动画绑定、技能资产、参数索引和关键技能语义复刻结果；任一项失败都必须使该角色验证失败，禁止降级为仅警告
